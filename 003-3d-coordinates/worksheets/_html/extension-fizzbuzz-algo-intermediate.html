If the block's number… it says colour
divides by 3 and 5 FIZZ BUZZ 🟡 gold
divides by 3 FIZZ ⬛ black
divides by 5 BUZZ 🟪 purple
divides by neither (nothing) 🟩 green

🔢 Big idea: a block's colour comes from its number. The code divides the number by 3 and by 5 and looks at the remainder (what's left over). Remainder 0 means it divides evenly.

2 · Spot the Bug 🐛

Bug A — the order is upside down. Gold should win when a number divides by 3 and 5. But the ÷ 3 check runs first and grabs block 15 before the gold check.

if remainder of N ÷ 3 = 0:
    paint BLACK    ← FIZZ
else if remainder of N ÷ 3 = 0 and remainder of N ÷ 5 = 0:
    paint GOLD     ← FIZZ BUZZ

What colour does block 15 wrongly get? Circle one: gold · black · purple

Which check must go first to fix it? Circle one: the ÷ 3 check · the ÷ 3 and ÷ 5 check

Bug B — wrong divisor. The purple BUZZ stripe should mark blocks that divide by 5, but this line uses 4.

else if remainder of N ÷ 4 = 0:
    paint PURPLE   ← BUZZ

Fill the blank to fix it:

else if remainder of N ÷ __ = 0:
